## 1. 登录认证基线

```
场景: auth-login.js  |  5 VU × 30s = 580次登录

  P50:     258ms
  P95:     263ms        ← 阈值<200ms ❌ (BCrypt=10)
  P99:     280ms
  吞吐:    19.3 req/s
```

## 3. 容量估算

```
当前配置 (单节点 Docker · 4C8G):
  登录: ~19 req/s  |  混合: ~13 req/s
  100并发登录时 QPS ≈ 19 × (100/5) ≈ 380 login/s
  login 瓶颈：BCrypt → CPU密集型，4核理论上限 ~80 login/s
  
建议:
  · <500仪表工厂: 当前配置足够 (P95 <15ms)
  · 500-2000仪表: 数据库连接池 50→100, JVM heap 512M→1G
  · >2000仪表: InfluxDB 独立节点, 应用节点×2 + Nginx负载均衡

预估拐点:
  查询吞吐: ~500 req/s (JVM线程池128 + 数据库连接池50)
  登录吞吐: ~60 req/s (BCrypt cost=10 · 4核上限)
  内存: 512MB heap 足以撑 1000仪表查询
```
