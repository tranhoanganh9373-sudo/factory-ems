### 3.2 响应体大小分布


### 3.3 登录耗时与安全权衡

```
BCrypt cost=10 → 238ms/次

攻击面分析:
  30次登录 × 238ms = 7.14s (无他延迟)
  字典攻击1000次 ≈ 4分钟 (无防护)
  字典攻击10000次 ≈ 40分钟 (无防护)

当前: 无限速 → 攻击者可自动化
修复后: 10次/分钟 → 攻击者需间断性尝试，10000次需~17小时
```

## 四、API 契约一致性分析

### 4.1 响应格式一致性

```
所有已验证端点响应格式: {code, data, message, traceId}
  ✅ 100%一致性

code域语义分布:
  code=0: 成功 (30个端点)
  code=400: 参数错误 (4个端点)  
  code=40001: 业务未授权/认证失败 (5个端点)
  code=40004: 资源未找到 (3个端点)

错误码分类合理，前端可基于code做统一错误处理。
```

### 4.2 参数兼容性问题

| 接口 | 问题 | 影响 |
|------|------|------|
| /dashboard/kpi | energyType=ELEC返回40001而非空数组 | 前端下拉筛选异常 |
| /report/ad-hoc | 需要from参数，但UI文档未明确标注 | 报表导出报错 |
| /bills | 缺periodId返回400而非分页列表 | 与其他list接口不一致 |
