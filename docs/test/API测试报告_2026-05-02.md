## 四、安全测试

| 测试项 | 结果 | 风险等级 |
|-------|------|---------|
| 无Token访问 | ✅ 401 拒绝 | - |
| 伪造Token访问 | ✅ 401 拒绝 | - |
| SQL注入 | ✅ 200 未崩溃 | - |
| 不存在ID | ✅ 404 | - |
| 参数校验 | ✅ 400 + 具体字段 | - |
| **登录限流** | ❌ 30次连续登录无拦截 | 🔴 HIGH |
| BOLA (水平越权) | 未测 | 🟡 MED |
| Token刷新 | 未测 | 🟡 MED |
| 密码锁定(5次) | 未测 | 🟡 MED |

**严重发现：登录端点无频率限制。** 连续30次 POST /auth/login 全部返回200，无429限流、无临时锁定。结合现有 BCrypt cost=10（238ms/次），攻击者可用慢速字典攻击。

建议：在 RateLimitFilter 中将 `/auth/login` 纳入限流规则（如 10次/分钟/IP）。
