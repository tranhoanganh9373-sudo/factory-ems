# 现场安装 SOP（电表 + 串口服务器）

> **受众**：现场电气施工 / 弱电施工 / 调试工程师
> **前置**：物料按 [meter-selection-guide.md](./meter-selection-guide.md) 备齐
> **后置**：施工完毕后按 [channel-config-import.json](./channel-config-import.json) + `scripts/import-channels.sh` 录入 EMS

---

## 0. 安全前置（每次进现场必读）

- [ ] **强电作业必须停电 + 验电 + 挂牌**（GB 26860 / DL 408）
- [ ] 至少 **2 人作业**，1 人监护
- [ ] 工具齐全：万用表、压线钳、热缩管、绝缘胶带、笔记本电脑、串口调试助手、网线、Modbus Poll 软件
- [ ] 与车间生产对口确认作业窗口（避免影响生产）

---

## 1. 工艺流程总览

```
① 现场踩点  →  ② CT 安装  →  ③ 电表安装  →  ④ 485 走线  →  ⑤ 串口服务器上电
                                                                   ↓
⑩ 联调验收  ←  ⑨ EMS 录入  ←  ⑧ 串口服务器联网  ←  ⑦ 单表点表测试  ← ⑥ 总线检测
```

每楼层独立施工，互不影响生产时段错开。

---

## 2. 步骤 ①：现场踩点（开工前 1 周）

| 任务 | 输出 |
|---|---|
| 测量每楼层配电室到 EMS 机房的网线距离 | 网线总米数 |
| 确认每个配电柜内有无 DIN 导轨富余位置 | 串口服务器安装位 |
| 拍照每个待装电表的回路（铭牌、断路器、空间） | 现场图集 |
| 与车间确认每条回路的负荷类型与功率 | CT 变比最终确认 |
| 与厂网管确认 IP 段、VLAN、是否需要二层隔离 | IP 分配表 |

**输出文档**：填好的 `meter-register-mapping-template.csv` 前 5 列（floor / panel / meter_tag / meter_model / unit_id），交给采购确认 CT 数量。

---

## 3. 步骤 ②：CT 安装（电气班）

- 一次侧穿芯方向：**P1 → P2 朝向负载**（接反会导致功率为负）
- 二次侧 S1 / S2 接电表对应电流端子
- **二次侧严禁开路**（穿芯但电表未接 → 高压击穿，可能伤人）
- 改造类项目优先用开口式 CT，安装时打开夹住一次电缆，再用螺丝锁紧

**验收**：钳形表测一次电流 vs 电表二次×变比，误差 < 2% 算合格。

---

## 4. 步骤 ③：电表安装

| 子步骤 | 操作 |
|---|---|
| 4.1 | 嵌入式表装在柜门 92×92 开孔；导轨表卡在 35 mm DIN 导轨 |
| 4.2 | 接 220 V 辅助电源（L / N），上电观察显示屏自检 |
| 4.3 | 接 CT 二次（A/B/C 相分别接对应电流端子，注意极性） |
| 4.4 | 接电压采样（A/B/C/N 接对应电压端子，相序勿错） |
| 4.5 | **设置站号**：按面板菜单 → "通讯参数" → unit_id 改为本表分配的唯一站号（同一总线不能重号） |
| 4.6 | **设置波特率**：统一改为 9600，8N1 |
| 4.7 | 在电表外壳贴标签：`<meter_tag>  unit_id=<x>` |

> **关键**：第 4.5 步如果忘改，所有表都是默认 unit_id=1，总线上一上电就全冲突，整条线全废。**改完一台测一台**。

---

## 5. 步骤 ④：RS-485 走线

### 5.1 物理拓扑（手拉手 / Daisy chain）

```
[串口服务器] ── A ── [表1 A] ── A ── [表2 A] ── ... ── [表N A] ── 120Ω ── B（终端）
              └─ B ── [表1 B] ── B ── [表2 B] ── ... ── [表N B] ── 120Ω ──/
```

### 5.2 接线规范

- 用 **RVSP 2×0.75 mm² 屏蔽双绞线**（正反方向必须一致：A 接 A、B 接 B）
- **手拉手串接**，不允许星型或分叉（分叉 > 30 cm 会反射）
- 每段两端都剥屏蔽层，**屏蔽层只在一端接地**（通常在串口服务器侧，避免地环流）
- **总线两端各加一个 120 Ω 终端电阻**（在第一台和最后一台电表的 A-B 之间）
- 最长 1000 m @ 9600（超过加 RS-485 中继器）

### 5.3 标识

每段电缆两端贴标签：`<from>-<to>  楼层-总线编号`

---

## 6. 步骤 ⑤：串口服务器上电与配置

| 子步骤 | 操作 |
|---|---|
| 6.1 | DIN 导轨安装；接 24 VDC 或 220 V（看型号） |
| 6.2 | 网线接到楼层交换机（不要接到办公网） |
| 6.3 | 笔记本同段 IP 通过 Web UI 登录（USR-N510 默认 192.168.0.7，admin/admin） |
| 6.4 | 改设备 IP 为厂网管分配的固定 IP（如 10.0.1.11） |
| 6.5 | 串口参数设置：**9600 / 8 / 1 / N / 无流控** |
| 6.6 | 工作模式：**TCP Server**，本地端口 502 |
| 6.7 | 包间隔：保留默认（一般 50 ms） |
| 6.8 | 保存重启，确认指示灯：电源常亮、Link 常亮、Active 闪烁 |

**验收**：笔记本 `telnet 10.0.1.11 502` 能连通即可。

---

## 7. 步骤 ⑥：总线检测（关键调试节点）

### 7.1 物理层检测（断电）

| 项 | 期望值 | 不合格处理 |
|---|---|---|
| A-B 之间电阻（断电） | **60 Ω 左右**（120 Ω∥120 Ω） | 远高 → 终端电阻没装 / 接错；远低 → 短路 |
| A-地、B-地 | 高阻（> 1 MΩ） | 低阻 → 屏蔽层接错 |

### 7.2 通讯层检测（上电）

笔记本装 **Modbus Poll**（或 Linux 用 `mbpoll` 命令行工具）：

```bash
# 通过串口服务器测每台表
mbpoll -m tcp -a <unit_id> -r 1 -c 10 -t 4 10.0.1.11
#  -a: unit_id   -r: 起始寄存器  -c: 数量  -t: 4 = HOLDING
```

逐个 unit_id 测试：
- 全部应答 → ✅ 总线 OK
- 部分超时 → 检查那块表的接线 / 站号 / 终端电阻
- 全部超时 → 极性反了（A-B 互换试试）

---

## 8. 步骤 ⑦：单表点表测试

用 Modbus Poll 按 [meter-register-mapping-template.csv](./meter-register-mapping-template.csv) 逐行验证：

- 电压寄存器读出来 ≈ 现场实际电压（万用表对照）
- 电流寄存器 ≈ 钳形表读数 × CT 变比
- 功率不应为负（为负说明 CT P1/P2 反了）
- 电量寄存器读两次，间隔 1 分钟，差值 / 间隔 ≈ 当前功率（验证累计电量正确）

异常处理速查：

| 现象 | 原因 | 处置 |
|---|---|---|
| 电压读出来是 380 但实际 220 | 接成相间电压了 | 改接相电压端子 |
| 电流为 0 但实际有负荷 | CT 二次没接 / S1 S2 接反 | 复查 CT 接线 |
| F32 数值是天文数字 | byte_order 不对 | 改 ABCD ↔ CDAB ↔ BADC ↔ DCBA 试 |
| 功率为负 | CT P1/P2 装反 | 软件改 CT 方向 或 拆 CT 换向 |

---

## 9. 步骤 ⑧：串口服务器联网验证

EMS 服务器侧（机房）：

```bash
# 从 EMS 机器测每台串口服务器
for ip in 10.0.1.11 10.0.1.12 10.0.1.13 10.0.1.14; do
  echo "Testing $ip:502 ..."
  nc -zv $ip 502 -w 2
done
```

全部 `succeeded!` 即可进入 EMS 录入阶段。

---

## 10. 步骤 ⑨：EMS 录入

按 [channel-config-import.json](./channel-config-import.json) 走批量导入，或参考 `scripts/import-channels.sh`。

录入后在 `/collector` 页：
- 4 条通道全部 `CONNECTED` → 绿
- 每条通道点「测试」→ latencyMs < 200 算正常

---

## 11. 步骤 ⑩：联调验收（≥ 24 小时观察）

| 验收项 | 通过标准 |
|---|---|
| 24h 通讯成功率 | **≥ 99%**（< 95% 看哪台表掉队） |
| 电压/电流读数与现场万用表 | 误差 < 2% |
| 整点电量增量 | 与上小时功率 × 1h 误差 < 5% |
| 拔掉一根 485 线后 EMS 状态 | 5 分钟内变 ERROR + 触发告警 |
| 重新插上 EMS 状态 | 1 分钟内自动恢复 CONNECTED |

**验收文档**：填一份《现场验收记录表》归档：项目名、楼层、表号、安装日期、施工人、调试人、24h 成功率截图。

---

## 12. 现场常见问题速查

| 现象 | 排查路径 |
|---|---|
| 一条总线全部超时 | ① 极性 ② 终端电阻 ③ 串口服务器串口参数 ④ 串口服务器有没有真上电 |
| 单台表偶尔超时（< 5%） | 接触不良 / 端子松动 / 屏蔽干扰 / 该表故障 |
| 单台表持续超时 | 站号冲突 / 这台表波特率没改 / 这台表硬件坏 |
| 全部功率为负 | 整段 CT 反了，软件改方向最快 |
| 偶发数据跳变（电压瞬间到 99999） | byte_order 偶尔错位（不严重）/ 仪表瞬时干扰；可在 EMS 加 `clamp` 规则 |
| EMS 一切正常但 24h 后告警「数据漂移」 | 电量寄存器溢出 / 累计字段断电后清零 → 找仪表手册看溢出处理策略 |

---

## 13. 维护建议

- 每季度看一次 24h 成功率报表，发现下降立即排查
- 每年一次屏蔽层接地阻抗复测（雷电季节后必测）
- 串口服务器每 2 年检修（电源电容老化是主要风险）
- 备件库存：1–2 台同型号串口服务器 + 5 台同型号电表
