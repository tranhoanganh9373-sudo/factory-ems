# 电价（ems-tariff）· 功能概览

> 适用版本：v1.3.0+ ｜ 最近更新：2026-05-01
> 受众：销售 / 客户 / 财务 / 实施工程师

---

## §1 一句话价值

把电网公布的复杂电价规则（尖峰平谷分时 + 阶梯 + 季节切换 + 容量电费）一次性录入系统，账单和报表自动按当时生效的电价取价。换季、换价、换政策时，改一处生效全平台，不用人工对账电价表。

---

## §2 解决什么问题

- **电价表录在 Excel 公式里，每次政策变动改起来如履薄冰**：一个时段单价错了，整月账单全错；财务每月对一次电网账单时人工反向核算电价。
- **分时电价规则复杂**：尖峰平谷的时段不仅一年两换（夏冬季时段不同），政策每年还可能调整，不能写死在代码里。
- **历史账单要保留当时电价**：今年的账单按今年电价算，去年账单要保留按当时电价算的口径，否则查历史变成"重新出账"。
- **多套场地适用不同电价**：公司有多个分厂区，每个厂区电网公司不同，电价方案差异大。

---

## §3 核心功能

- **电价方案（plan）+ 时段（period）两层结构**：
  - **方案**：一份电价表，绑定能源类型、生效起止日期。
  - **时段**：方案下的多条规则，每条定义"周一到周日的某时段适用哪种价位"。
- **4 种时段类型**（`period_type`）：尖峰（SHARP）/ 峰（PEAK）/ 平（FLAT）/ 谷（VALLEY）。每种可对应不同单价。
- **生效期管理**：每个方案有 `effective_from` / `effective_to`，到日期 0:00 自动生效，旧方案进入历史可查。
- **跨节点适用**：方案不绑定到组织节点——用 "resolve" 接口查询"某节点 + 某时间点 + 某能源类型 → 当时适用什么单价"，业务层在算账时调用。
- **分时段单价**：每个时段一行，存周几（0-6 / 1-7）+ 起止时分秒 + 单价（分 / 元 / 元每度）。
- **乐观锁 + 启用开关**：方案带 `version` 防覆盖，`enabled` 字段可暂停某方案。
- **API 解析端点**（`GET /api/v1/tariff/resolve`）：账单和报表用此端点获取"当时电价单价"，避免业务逻辑各自实现取价。

## §4 适用场景

### 场景 A：实施工程师 — 录入新部署的电价

客户在江苏，按"工商业及其他用电"两部制电价。实施工程师拿到当地供电公司的电价表（含 4 个时段、夏冬切换、容量电价）：

1. 进 `/tariff`，新建方案"江苏-2026 工商业用电"，能源类型 = 电，生效起 2026-01-01。
2. 录入夏季时段（6-9 月）：尖峰 11:00-13:00、19:00-21:00 等共 4-6 行。
3. 录入冬季时段（10-5 月）：每行单独。
4. 容量电价 / 力调电价（v2.x 起）单独配。
5. 保存 → 账单引擎下次出账自动取这份方案。

### 场景 B：电价调整 — 春节后涨价

电网公告 2026 年 4 月 1 日起调整。财务和管理员在 3 月底进 `/tariff`：

1. 复制现有方案为"江苏-2026.4 工商业用电"，生效起 2026-04-01。
2. 在新方案上改各时段单价。
3. 把旧方案的 `effective_to` 设为 2026-03-31。
4. 4 月 1 日 0:00 自动生效，4 月账单按新价出，3 月及之前账单按旧价出。

### 场景 C：多场地适配 — 按节点指定方案

公司在江苏和浙江各有一个厂区，两地电价不同。在每个组织节点上配置该节点适用的方案 ID（一对多关系扩展，v2.x 起内置；v1.x 通过外部映射表实现）。账单引擎按节点 → 找方案 → resolve 单价 → 出账。

## §5 不在范围（首版）

- **不做需求侧管理（DSM）**：不能基于实时负荷动态调价，电价规则是静态预设。
- **不做现货电价 / 实时电价**：仅支持分时（按时段）+ 阶梯（按累计用量）静态规则；现货电价（按 5 分钟实时电网价格）需定制。
- **不做容量电价自动核算**：容量电费通常按合同需量或最大需量计算；首版只存"容量电价单价"作为参考字段，月度自动计算容量费用规划在 v2.x 起。
- **不做力调电费**：功率因数与力率电费的奖励/惩罚需要采集层支持功率因数实时数据，规划中。
- **不做电价审计签字流程**：方案修改直接生效，重要变更靠审计日志追溯。
- **不做用户分类（居民 / 商业 / 工业）联动**：用户可以建多个方案对应不同分类，但不在系统里强约束。

## §6 与其他模块的关系

```
ems-orgtree            可在节点上配置该节点适用的电价方案
       │
       ▼
ems-tariff（电价方案 + 时段）
       │
       ├──────> ems-billing       账单按 (节点, 时间, 能源类型) → resolve 单价
       ├──────> ems-cost          分摊后的账单也走电价
       ├──────> ems-report        分时电价分布报表（哪个时段电费占大头）
       └──────> ems-dashboard     /tariff-distribution 端点展示价位分布饼图
```

电价是计费的核心元数据，账单 / 报表 / 仪表盘都依赖它。

## §7 接口入口

- **前端路径**：`/tariff`
- **API 前缀**：`/api/v1/tariff`
- **关键端点**：
  - `GET /plans` — 方案列表（含筛选：能源类型、是否启用、生效日期）
  - `GET /plans/{id}` — 方案详情含所有时段
  - `POST /plans` / `PUT /plans/{id}` — 新建 / 修改方案
  - `DELETE /plans/{id}` — 删除（被账单引用时拒绝；建议改为禁用）
  - `GET /resolve?energyTypeId=X&at=2026-05-01T10:30:00` — 业务侧调用，返回当时单价

## §8 关键字段

### `tariff_plans` 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint | 主键 |
| `name` | varchar | 方案名 |
| `energy_type_id` | bigint | 能源类型 |
| `effective_from` | date | 生效起 |
| `effective_to` | date | 生效止（含），可空表示永久 |
| `enabled` | bool | 启用开关 |
| `version` | bigint | 乐观锁 |
| `created_at` / `updated_at` | timestamp | |

### `tariff_periods` 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint | 主键 |
| `plan_id` | bigint | 所属方案 |
| `period_type` | varchar | `SHARP` / `PEAK` / `FLAT` / `VALLEY` |
| `time_start` | time | 起始时分秒（如 `08:00:00`）|
| `time_end` | time | 结束时分秒（如 `11:00:00`）|
| `price_per_unit` | numeric(12,4) | 单价（元 / 度，4 位小数）|
| `created_at` | timestamp | |

> 时段表当前不带"周几 / 季节"字段——通过多条记录组合表达。如需季节切换，建多个方案，分别设 effective_from / to。
