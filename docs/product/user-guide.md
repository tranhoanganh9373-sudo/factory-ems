# Factory EMS · 用户使用手册（总览）

> 适用版本：v1.7.0+ ｜ 最近更新：2026-05-01
> 受众：最终用户 — 管理员（ADMIN）、运维（OPERATOR）、财务（FINANCE）、查看者（VIEWER）
> 阅读时长：按章节挑读，全篇约 25 分钟

本文档是**平台总览版**用户手册，按角色组织日常操作场景。

子功能的详细操作步骤见同目录下：
- [`alarm-user-guide.md`](./alarm-user-guide.md) — 告警操作详细步骤

## §1 角色与权限速查

平台预置 4 个角色。一个用户**可以同时拥有多个角色**，权限取并集。

| 角色码 | 名称 | 看 | 做 |
|---|---|---|---|
| `ADMIN` | 管理员 | 全部 | 全部（含用户管理、电价、采集配置、关账解锁、Webhook）|
| `OPERATOR` | 运维 | 仪表盘 / 报表 / 告警 / 平面图 | 处理告警、开关维护模式 |
| `FINANCE` | 财务 | 仪表盘 / 报表 / 账单 / 分摊 | 配分摊规则、跑分摊、关账 |
| `VIEWER` | 查看者 | 按节点权限读 | 无写操作（只能改自己的密码 / 资料）|

**节点级权限（除 ADMIN 外）**：用户绑定到组织节点的子树，看不到子树外的数据。
- 财务 A 绑定到"主厂区" → 只看到主厂区下的账单 / 报表
- 运维 B 绑定到"二车间" → 只看到二车间下的告警

ADMIN 不受节点限制，看全树。

## §2 第一次登录

### 2.1 拿到地址和初始密码

实施工程师交付后，你应该拿到：
- **平台地址**：例如 `http://10.0.0.10:8888` 或 `https://ems.your-company.com`
- **初始用户名 / 密码**：管理员通常是 `admin` / `admin123!`，业务用户由管理员创建后下发

### 2.2 首次登录

1. 浏览器打开平台地址，看到登录页
2. 输入用户名 / 密码，点"登录"
3. 系统强制进入"修改密码"流程（首次登录不修改密码无法继续）
4. 设置新密码：8 位以上，含大小写字母 + 数字 + 符号
5. 进入仪表盘（首页）

[截图：登录页 + 首次修改密码弹窗]

### 2.3 顶部导航与角色徽标

- 左上：平台 logo + 系统名
- 中间：主菜单（按角色显示不同条目，看不到的不代表不存在，而是没权限）
- 右上：铃铛角标（告警通知，OPERATOR / ADMIN 可见）+ 用户菜单
- 用户菜单：当前用户名、角色、修改密码、退出登录

### 2.4 修改个人资料

进入 `/profile`：
- 修改密码（输入旧密码 + 两次新密码）
- 修改显示名称 / 联系方式

## §3 ADMIN（管理员）日常

### 3.1 用户与角色管理

**入口**：`/admin/users`、`/admin/roles`

**常见任务**：

#### 创建用户
1. 进 `/admin/users`，点"新建"
2. 填写用户名、初始密码、显示名、所属角色（可多选）
3. 配置节点权限范围（财务 / 运维 / 查看者必选）
4. 保存 → 把用户名 + 初始密码下发给该用户

#### 重置用户密码
1. 在用户列表搜索用户名
2. 点"重置密码"，系统生成临时密码
3. 把临时密码告知用户（用户首次登录后会被强制改密）

#### 临时禁用用户
1. 用户列表 → 编辑 → "状态"切到"禁用"
2. 该用户立即无法登录（已登录会话在下一次接口调用时失效）

#### 查角色权限矩阵
进 `/admin/roles`，看每个角色拥有的操作码。**角色权限码由产品定义，不可自定义**。

[截图：用户列表 + 编辑用户对话框]

### 3.2 电价管理

**入口**：`/tariff`

**常见任务**：

#### 录入新电价
1. 拿到电网公布的最新电价表（含分时段单价、容量电价等）
2. 进 `/tariff`，点"新建电价方案"
3. 填写：方案名称、生效日期、分时段单价（峰 / 平 / 谷 / 尖峰）、容量电价
4. 保存，到生效日期 0:00 自动启用，旧电价进入历史

#### 同时配置多个方案（按节点）
不同分厂区可以适用不同电价方案。在每个组织节点上指定该节点适用的方案 ID。

#### 查电价历史
账单出错时常见排查路径：进电价历史，按账单期次时间核对当时生效的方案。

[截图：电价方案列表 + 新建方案对话框]

### 3.3 采集配置

**入口**：`/collector`（看状态）+ 服务器上的配置文件（改参数）

**常见任务**：

#### 看采集健康
进 `/collector`，看每个通道、每台设备的：
- 在线 / 离线状态
- 最近一次成功时间
- 采集成功率（最近 1 小时 / 24 小时）
- 缓冲区积压条数

#### 新增设备到采集
1. 先在 `/meters` 登记仪表元数据（名称、节点、能源类型、采集协议、设备地址）
2. 在采集配置文件（`config/collector.yml` 或类似路径）追加该设备的轮询配置
3. 重载配置或重启采集服务

详细步骤见 [`collector-protocols-user-guide.md`](./collector-protocols-user-guide.md)。

### 3.4 告警 Webhook 配置

**入口**：`/alarms/webhook`（仅 ADMIN）

**常见任务**：

#### 接钉钉机器人
1. 在钉钉群创建一个自定义机器人，拿到 Webhook URL
2. 进 `/alarms/webhook`，填入 URL + 共享密钥（用于 HMAC 签名验证）
3. 点"测试"，钉钉群应该收到一条测试消息
4. 保存

#### 接企业微信机器人
同上，把 URL 换成企业微信群机器人地址。

#### 排查 Webhook 失败
进 `/alarms/webhook`，下方"下发流水"列表显示每次推送的状态码 + 响应。失败的会自动重试，重试都失败会进入失败队列。

详细见 [`alarm-webhook-integration.md`](./alarm-webhook-integration.md)。

### 3.5 审计与合规

**入口**：`/admin/audit`

**常见任务**：

#### 查"谁改了什么"
按用户、模块、动作、时间范围筛选。例：
- 查上周谁改过电价 → 模块 = tariff，动作 = update
- 查某个仪表的阈值变更历史 → 搜索关键词 = 仪表 ID

#### 导出审计日志
点列表右上"导出"，导出 CSV，用于合规归档。

#### 关账后解锁审计
财务关账后再解锁的操作会被特别标记，便于审查。

### 3.6 关账期解锁

**入口**：`/bills/periods`（ADMIN + FINANCE）

如果财务关账后发现账单错误，ADMIN 可临时解锁让财务修正。流程：
1. 财务先关账，状态 = `LOCKED`
2. ADMIN 进 `/bills/periods`，点该期次"解锁"
3. **必填备注**：写明解锁原因（如"修正 6 月分摊规则错误"）
4. 系统进审计日志
5. 财务修正后再次关账

## §4 OPERATOR（运维）日常

### 4.1 看采集健康总览

**入口**：`/alarms/health`

进入后看到：
- 全场设备的在线 / 离线汇总
- 最近 7 天告警最多的 Top 10 设备
- 当前活跃告警列表

每天上班第一件事：扫一眼此页，确认无大面积红点。

[截图：alarms/health 页面，显示在线率饼图 + Top 10 列表]

### 4.2 处理一条告警

#### 收到告警
- **站内**：右上铃铛出现红点 + 数字角标
- **外推**：钉钉 / 企业微信群收到推送（如果配过 Webhook）

#### 处置流程
1. 点铃铛 → 通知抽屉打开 → 看到告警条目（设备名 + 触发时间 + 触发原因）
2. 点告警 → 跳到 `/alarms/history` 详情页
3. 看"触发原因"（静默超时 / 连续失败）和"已持续时长"
4. 决定行动：
   - **现场处置**：派人去现场恢复网线 / 重启网关 / 检查仪表
   - **维护模式**：如果是计划停机，开维护模式后告警自动屏蔽
   - **暂时确认**：点"已确认"，状态从 `TRIGGERED` 转 `ACKNOWLEDGED`，但告警不消失
5. 数据流恢复后，告警自动转 `RESOLVED`，铃铛角标减 1
6. 如果数据流不会自然恢复（如设备已永久退役），手动点"关闭"告警

详细操作见 [`alarm-user-guide.md`](./alarm-user-guide.md)。

### 4.3 维护模式

#### 开维护模式（提前规划）
1. 计划停机前 30 分钟，进 `/meters`
2. 找到要维护的设备，点"开启维护模式"
3. **必填备注**：写明维护原因（如"6 月 15 日年度检修"）
4. 系统进审计，告警引擎自动跳过该设备

#### 关维护模式
1. 维护完成，进 `/meters`，点"关闭维护模式"
2. 必填备注（如"年检完成，恢复采集"）
3. 系统恢复正常监测

**坑点**：忘记关维护模式 = 真断线无告警。建议加一条"维护模式持续 > 3 天 → 提醒"的运维规则（v2.x 起内置）。

### 4.4 实时巡检（平面图）

**入口**：`/floorplan`

- 选择底图 → 看到该底图上所有挂点
- 绿色 = 在线、黄色 = 静默警告、红色 = 告警 / 离线
- 鼠标悬停看实时读数

每天巡检看一遍每张底图，能发现局部异常（如某个区域多个仪表同时变黄 = 可能局部网络问题）。

## §5 FINANCE（财务）日常

### 5.1 月初打开新期次

**入口**：`/bills/periods`

#### 流程
1. 月初（如 6 月 1 日）进 `/bills/periods`
2. 上月（5 月）期次状态应该是 `OPEN` 或 `CLOSED`
3. 点"新建期次"，选择月份 = 6 月
4. 期次状态 = `OPEN`，可开始接收账单数据

### 5.2 配置 / 调整分摊规则

**入口**：`/cost/rules`

#### 新建分摊规则
1. 点"新建规则"
2. 选择源节点（如"主厂区总进线"）+ 目标节点列表（如"一车间 / 二车间 / 办公区"）
3. 选择分摊方式：
   - **按面积**：按目标节点的面积字段比例
   - **按人头**：按目标节点的人数字段比例
   - **按产量**：按上月各节点产量数据比例
   - **自定义比例**：手动指定每个节点的百分比
4. **试算（dry-run）**：保存前一定先试算，看每个节点拿多少
5. 确认无误，保存规则

#### 调整规则
- 编辑现有规则，**调整后必须重新试算**
- 已应用过该规则的历史账单不受影响（除非解锁重跑）

### 5.3 跑月度分摊

**入口**：`/cost/runs`

1. 月初（账单出之前），点"新建分摊运行"
2. 选择期次（如 5 月）
3. 系统异步跑（大型规则可能需要几分钟）
4. 跑完进 `/cost/runs/<id>` 看明细：每条记录"哪个节点 / 多少能耗 / 多少金额 / 用了哪条规则"
5. 如有疑问，导出明细 Excel 复核

### 5.4 出月度账单

**入口**：`/bills`

1. 分摊跑完后，账单引擎自动按"用量 + 电价 + 分摊"出账单
2. 在 `/bills` 列表看本期所有账单（每个组织节点一张）
3. 点进账单 → 看明细（按时段、按电价档位拆分）
4. 复核无误，进 `/bills/periods` 点该期次"关账"

[截图：账单详情页 - 展示节点名、期次、总额、明细行]

### 5.5 关账与归档

**关账后**：
- 账单进入 `LOCKED` 状态，不可改
- 历史报表正常可查
- 如需修改，找 ADMIN 解锁（见 §3.6）

**关账前 checklist**：
- [ ] 所有节点账单都已生成（无缺失）
- [ ] 大额账单已抽样复核
- [ ] 分摊明细与去年同期对比无异常波动
- [ ] 客户已收到草稿确认

### 5.6 月报导出

**入口**：`/report/monthly`

- 选月份 + 节点 → 点"生成"
- 后台异步跑，跑完出现"下载"按钮
- 下载 Excel 给客户 / 内部归档

## §7 跨角色协作流（贯通）

### 7.1 一次断线引发的连锁反应

```
设备断线
   │
   ▼  (5 秒内)
   ems-alarm 检测到 → 进入 TRIGGERED 状态
   │
   ├─→ 站内通知    OPERATOR / ADMIN 收到铃铛
   ├─→ Webhook 推  钉钉群 / 企微群收到消息
   └─→ 审计日志    记录触发事件
   │
   ▼  (OPERATOR 介入)
   OPERATOR 派人现场处置
   │
   ▼
   设备恢复，数据流入
   │
   ▼  (自动)
   ems-alarm 检测到恢复 → 进入 RESOLVED 状态
   │
   ▼
   月底
   │
   ├─→ FINANCE 出账单时   该设备的数据完整可用
   └─→ 报表无空洞         能耗趋势平滑
```

**协作要点**：
- ADMIN 提前配好 Webhook → OPERATOR 才能在第一时间收到
- OPERATOR 提前开维护模式 → 计划停机不出噪音
- FINANCE 月底关账前 → 看一眼告警历史，确认当月数据完整

### 7.2 一次电价调整的协作

```
电网公告新电价（6 月 1 日生效）
   │
   ▼  (5 月底)
   ADMIN 进 /tariff 录入新电价方案
   │
   ▼
   FINANCE 在 /bills 选 5 月草稿账单 → 切换到新电价试算 → 复核
   │
   ▼
   到 6 月 1 日 0:00
   │
   ▼  (自动)
   新电价生效
   │
   ▼  (7 月初)
   FINANCE 出 6 月账单 → 自动用新电价
```

### 7.3 一次组织调整的协作

```
公司新增"三车间"
   │
   ▼
   ADMIN 进 /orgtree 在"主厂区"下加节点"三车间"
   │
   ▼
   ADMIN 把新仪表登记到 /meters（节点选三车间）
   │
   ▼
   ADMIN 配采集 → 三车间数据开始流入
   │
   ▼
   FINANCE 在 /cost/rules 调整分摊规则（一车间 / 二车间 / 三车间）
   │
   ▼  (下次出账)
   账单自动包含三车间
```

## §8 常见问题（FAQ）

### Q1：登录后看不到某个菜单？
**A**：菜单按角色 + 权限码动态显示。看不到 = 没权限。找 ADMIN 检查你的角色是否包含相应操作码。

### Q2：仪表盘数据为空？
**A**：可能原因：
1. 节点筛选选错了 → 切换节点
2. 采集还没开始流入 → 进 `/collector` 看通道状态
3. 你的节点权限范围下没有任何在采的设备 → 找 ADMIN 检查

### Q3：报表生成很慢？
**A**：长跨度（年报）+ 大节点（含 200+ 仪表）首次生成慢正常。后续会用 rollup 加速。
**坑点**：如果 rollup 任务有失败窗口，相应区间可能逼回原始数据查询，会更慢。运维可在 `/admin/audit` 或可观测性看板上看是否有 rollup 失败堆积。

### Q4：账单金额对不上电网账单？
**A**：常见原因：
1. 电价方案录入错（峰平谷时段 / 单价）→ 在 `/tariff` 复核
2. 容量电费 / 力调电费未配
3. 分摊规则错（如总进线没有正确分摊到所有车间）→ 在 `/cost/rules` 复核
4. 仪表读数本身有缺失 → 在 `/alarms/history` 看当月有无未恢复告警

### Q5：忘记密码？
**A**：联系 ADMIN 重置（见 §3.1）。系统不开放邮件 / 短信自助找回（避免外发通道带来的攻击面）。

### Q6：我能不能临时让某个用户看一下别的部门的数据？
**A**：可以，但建议**新增一个角色** + 节点权限范围，而不是直接改原用户的范围。这样数据隔离边界清晰，审计也能查到"是谁、什么时候、为什么"开了这个权限。

### Q7：浏览器要求什么版本？
**A**：Chrome 100+、Edge 100+、Firefox 100+，推荐 Chrome 最新稳定版。不支持 IE。屏幕分辨率建议 1440×900 以上。

### Q8：手机能用吗？
**A**：当前版本未做移动端适配。手机看仪表盘可以（部分组件响应式），但平面图、报表、配置类页面在手机上不可用。移动端 App 在路线图中。

## §9 操作速查卡

打印此节贴在工位上。

### ADMIN 速查
| 任务 | 路径 | 频率 |
|---|---|---|
