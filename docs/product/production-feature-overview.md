# 班次与产量（ems-production）· 功能概览

> 适用版本：v1.3.0+ ｜ 最近更新：2026-05-01
> 受众：销售 / 客户 / 生产管理员 / 实施工程师

---

## §1 一句话价值

把工厂的班次（早 / 中 / 晚 / 夜班）和每班实际产量录入系统。能耗数据按班次窗口聚合，得出"单位产量能耗"这个能源 KPI。它回答的不是"用了多少电"，而是"每生产 1 件产品用了多少电"。

---

## §2 解决什么问题

- **能耗数据脱离生产**：知道工厂一天用 5000 度电，但不知道这 5000 度对应了多少产品产出，无法判断节能改造效果。
- **班次能耗对比靠手算**：早班 vs. 夜班谁更省电？三班产量不同，绝对值不能比，要算"单位产量能耗"才有意义。
- **绩效考核缺数据**：奖励节能班组、追责能耗高的班组，需要客观的"单位产量能耗"指标，不能靠经验判断。
- **节能改造效果难评估**：改造前后产量不同，绝对能耗变化无法直接对比，需要按单位产量能耗对比。

---

## §3 核心功能

- **班次定义（shifts）**：定义工厂的班次时段，例如：
  - 早班：8:00-16:00
  - 中班：16:00-24:00
  - 夜班：24:00-次日 8:00（跨日）
- **班次实例 + 产量录入（production_entries）**：每天每条产线 / 每个班次产生一条记录，运营人员在班结束后录入实际产量（件 / kg / 吨）。
- **批量导入**：支持 Excel 批量导入历史产量数据（`POST /production/entries/import`），方便从老系统迁移。
- **每日总产量汇总**：`GET /production/entries/daily-totals` 一次性拿某节点某时间段每日产量。
- **班次报表自动联动**：报表模块基于班次定义自动按班次窗口聚合能耗，得出"该班次能耗 / 该班次产量 = 单位产量能耗"。
- **跨日班次支持**：夜班从 22:00 跨到次日 6:00，系统自动处理时间窗口。
- **产线维度可选**：班次可以全厂统一（不分产线），也可以按产线区分（不同产线不同班次时段）。

---

## §4 适用场景

### 场景 A：制造工厂 — 标准三班制管理

一家钢铁加工厂三班倒。每天早 / 中 / 夜班结束后，班长在 `/production/entry` 录入本班次产量（如 80 吨钢卷）。系统自动按班次窗口聚合能耗（如该班 8 小时用了 12000 度电），得出单位产量能耗 = 12000 / 80 = 150 度/吨。月底报表对比三班的单位产量能耗 + 横向对比上月同期，找出能耗异常的班次。

### 场景 B：能源审计 — 单位产量能耗趋势分析

碳核查要交"单位产品综合能耗"指标，业务习惯按月报送。把过去 12 个月每月所有班次的产量录入（或一次性导入历史 Excel），月报自动出"每月单位产量能耗趋势曲线"，可直观看节能改造前后效果。

### 场景 C：节能改造效果评估

某车间装了变频器，改造前 1000 件产品用 5000 度电，改造后 1200 件产品用 5400 度电，绝对值看好像更费电，但单位产量能耗从 5.0 度/件降到 4.5 度/件，改造效果明显。这需要班次和产量数据支持。

## §5 不在范围（首版）

- **不是 MES**：不管工单、物料、库存、订单流转。只录"这班次产了多少件"这一个数据点。
- **不做产品种类细分**：每条产量记录只有一个数字 + 单位，不区分"产品 A 100 件、产品 B 200 件"。需要细分时建议按产品建多条产线（多组班次）。
- **不做实时产量采集**：当前版本产量靠人工录入或 Excel 导入；从 MES 自动同步规划在 v2.x。
- **不做单位换算**：录入的单位（件 / kg / 吨）由用户自己保证一致性，系统不做换算。
- **不做生产计划 / 排班管理**：班次时段是固定模板，不支持动态排班。
- **不做产量审批流**：录入后直接生效，错误数据靠审计日志追溯。

## §6 与其他模块的关系

```
ems-orgtree    班次和产量按节点（产线 / 车间）维度组织
       │
       ▼
ems-production
       │
       ├──────> ems-report       按班次维度聚合能耗 → /report/preset/shift
       ├──────> ems-dashboard    /energy-intensity 端点（单位产量能耗）
       └──────> ems-billing      高级用法：按产线产量分摊
```

班次定义是日 / 月 / 年之外的第四种时间维度，专门服务于工业能耗 KPI。

---

## §7 接口入口

- **前端路径**：
  - `/production/shifts` — 班次定义
  - `/production/entry` — 班次产量录入
- **API 前缀**：`/api/v1/shifts`、`/api/v1/production`
- **关键端点**：
  - `GET /api/v1/shifts` — 班次列表
  - `POST /api/v1/shifts` / `PUT /{id}` / `DELETE /{id}` — 班次 CRUD
  - `GET /api/v1/production/entries` — 产量录入列表
  - `POST /api/v1/production/entries` / `PUT /{id}` — 录入 / 修改
  - `POST /api/v1/production/entries/import` — Excel 批量导入
  - `GET /api/v1/production/entries/daily-totals` — 按日汇总产量

## §8 关键字段

### `shifts` 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint | 主键 |
| `name` | varchar | 班次名（"早班"/"中班"/"夜班"）|
| `time_start` | time | 起始时分（如 `08:00:00`）|
| `time_end` | time | 结束时分（如 `16:00:00`，可跨日）|
| `org_node_id` | bigint | 班次所在产线 / 车间 |
| `enabled` | bool | 启用开关 |

### `production_entries` 表

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint | 主键 |
| `shift_id` | bigint | 班次外键 |
| `org_node_id` | bigint | 产线 / 车间节点 |
| `entry_date` | date | 班次发生日期 |
| `quantity` | numeric | 产量数值 |
| `unit` | varchar | 单位（"件" / "吨" / "kg" / 自由文本）|
| `note` | varchar | 备注（如"质量问题报废 5 件"）|
| `created_at` / `updated_at` | timestamp | |
