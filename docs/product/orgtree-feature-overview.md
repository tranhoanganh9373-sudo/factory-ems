# 组织树（ems-orgtree）· 功能概览

> 适用版本：v1.1.0+ ｜ 最近更新：2026-05-01
> 受众：销售 / 客户 / 实施工程师

---

## §1 一句话价值

把工厂的组织层级（厂区 / 车间 / 产线）建成一棵树。所有报表、账单、报警、权限都按这棵树聚合或下放。树建好了，平台后续所有模块就都按这个层级运转。

---

## §2 解决什么问题

- **跨车间汇总靠手工**：报表选不了"主厂区下所有车间"，只能逐个车间相加，慢且容易漏。
- **权限切不细**：新人来了想给他"只看二车间"，但传统系统只有"全开"或"全关"两挡。
- **组织调整一动全乱**：新增车间或合并产线时，老报表 / 账单的口径跟着变，难以追溯历史。
- **分摊规则缺锚点**：按面积 / 按人头分摊需要存"每个节点多大、多少人"，没有专门的地方存。

---

## §3 核心功能

- **任意层级树结构**：根节点 → 子节点 → 子子节点……层数不限，前端用 antd `Tree` 组件展示，支持拖拽和搜索。
- **节点元数据**：每个节点存名称、编码（业务层用的稳定 ID）、节点类型、面积、人数、排序号。面积和人数是分摊规则的可选输入。
- **节点类型**：用 `node_type` 字段标识"厂区 / 车间 / 产线 / 部门 / 设备组"等业务语义，纯展示用，不影响功能。
- **乐观锁防覆盖**：每条记录带 `version` 字段，两个人同时编辑不会互相覆盖。
- **节点移动**：通过 `PATCH /{id}/move` 把子树搬到新父节点下，所有后代节点自动跟着搬。
- **稳定编码**：节点的业务编码（`code`）唯一且不可变，外部系统对接靠这个；显示名（`name`）可改。
- **软隔离**：禁用某节点不会删数据，只是用户看不到。

## §4 适用场景

### 场景 A：能源管理项目方 — 第一次给客户建组织

实施工程师拿到客户提供的组织架构表（Excel），按层级建树：根节点 = 客户公司名，下面挂"主厂区 / 分厂区"，主厂区下再挂"一车间 / 二车间 / 三车间"，每个车间下挂"产线 1 / 产线 2"。每个节点填上面积（用于按面积分摊）和人数（用于按人头分摊）。建完树，所有后续模块（仪表、报表、账单）都在这棵树上挂数据。

### 场景 B：物业 / 园区 — 按租户隔离

把每个租户作为根节点的子节点，下面挂该租户实际占用的楼层 / 房间。给每个租户用户绑定到自己的子节点，他们只看得到自己。新租户进来时加节点，老租户搬走时禁用节点（数据保留下来方便后续查询）。

### 场景 C：碳核查 / 双控 — 按工序聚合

碳核查机构常常要求按"工序"（铸造 / 加工 / 装配 / 包装）汇总能耗。把工序作为节点类型，把同工序的产线放在同一个父节点下，报表里选这个工序节点 → 自动汇总该工序的能耗。

## §5 不在范围（首版）

- **不做版本化历史**：节点改了名字，历史报表显示新名字；要保留改名前的历史名称需走审计日志查。
- **不做跨树挂接**：一个仪表只能挂在一个组织节点下，不支持"同一仪表同时属于两个组织"。
- **不做角色矩阵继承**：节点权限是"绑定到节点子树"，不支持复杂的"读 X 节点的某些字段、写 Y 节点的某些字段"权限矩阵。
- **不做组织变更工作流**：节点的新建 / 修改 / 删除直接生效，不带审批流。重要变更靠审计日志追溯。
- **不限制层级深度**：理论上可以建 100 层，但前端展示和查询性能在 6~8 层最佳。

## §6 与其他模块的关系

```
ems-orgtree（组织树）
   │
   ├──> ems-meter        每块仪表绑定到某个节点
   ├──> ems-floorplan    平面图按节点关联底图
   ├──> ems-tariff       电价方案可按节点指定
   ├──> ems-cost         分摊规则的源 / 目标都是节点
   ├──> ems-billing      账单按节点出，每个节点一张
   ├──> ems-report       所有报表的"维度"是节点
   ├──> ems-dashboard    所有 KPI 按节点过滤
   └──> ems-auth         非 ADMIN 用户绑定到节点子树
```

没有 orgtree 整个平台就跑不起来。组织树是第一个建、最稳定的元数据。

## §7 接口入口

- **前端路径**：`/orgtree`
- **API 前缀**：`/api/v1/org-nodes`
- **关键端点**：
  - `GET /tree` — 拉整棵树（一次性返回，前端缓存）
  - `GET /{id}` — 单节点详情
  - `POST /` — 新建节点
  - `PUT /{id}` — 编辑节点
  - `PATCH /{id}/move` — 移动子树
  - `DELETE /{id}` — 删除节点（有子节点或被引用时拒绝）

---

## §8 关键字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint | 主键 |
| `parent_id` | bigint | 父节点 ID，根节点为 NULL |
| `name` | varchar | 显示名称（可改）|
| `code` | varchar | 业务编码（唯一、不建议改）|
| `node_type` | varchar | 节点类型（"厂区"、"车间"、"产线" 等）|
| `area` | numeric | 面积（m²，分摊用，可空）|
| `headcount` | int | 人数（分摊用，可空）|
| `sort_order` | int | 同级排序 |
| `version` | bigint | 乐观锁版本号 |
| `created_at` / `updated_at` | timestamp | 审计字段 |
