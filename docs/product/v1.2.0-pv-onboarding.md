# v1.2.0 光伏接入升级说明

## 升级内容

新增「来源构成 GRID vs SOLAR」「PV 发电 vs 厂区负载曲线」「上网卖电收入抵扣」「碳排报表」四块功能。

## 旧账单 vs 新账单字段对比

| 字段 | v1.1.x | v1.2.0 |
|------|--------|--------|
| `amount` | 月度电费 | 月度电费（不变） |
| `feed_in_revenue` | — | **新增**：上网卖电收入（¥） |
| `net_amount` | — | **新增**：电费净额 = amount − feed_in_revenue |

## 「自发自用率」是什么

当光伏发电量超过厂区即时负载时，多余电会通过并网点回送电网（卖电）。

**自发自用 = max(0, hourly_pv_generation − hourly_grid_export)**，按小时累加；
自发自用率 = 自发自用 / 总发电量。

> 注意：必须先按小时取 `max(0, ...)` 再累加，**不能**先累加再 `max`。否则同一天内夜间的发电缺口会被白天的过剩抵消，得到偏高的自发自用率。

(前端面板 `PvVsLoadPanel` 直观展示双 Y 轴曲线。)

## 数据模型变更

### `meters` 表新增三字段（V2.6.0 migration）

| 列 | 类型 | 默认值 | 含义 |
|----|------|--------|------|
| `role` | VARCHAR(16) | `CONSUME` | 测点角色：`CONSUME` / `GENERATE` / `GRID_TIE` |
| `energy_source` | VARCHAR(16) | `GRID` | 能源来源：`GRID` / `SOLAR` / `WIND` / `STORAGE` |
| `flow_direction` | VARCHAR(16) | `IMPORT` | 流向：`IMPORT`（进口） / `EXPORT`（出口） |

老 meter 行升级后均落到默认值 → 行为完全等价于纯电网耗电。

### 新增基础数据表

| 表 | 用途 |
|----|------|
| `carbon_factor` | 各能源源（GRID / SOLAR / WIND / STORAGE）的碳排放因子（kg CO₂e / kWh），按区域 + 生效起始日期生效 |
| `feed_in_tariff` | 上网卖电定价（¥/kWh），按区域 + 能源源 + 时段类型（SHARP / PEAK / FLAT / VALLEY）生效 |

V2.6.0 已 seed 中国区域基础数据：GRID 0.5810、SOLAR 0.0480；SOLAR 上网价 0.42（FLAT 时段）。

### `bill` 表新增两列（V2.6.1 migration）

`feed_in_revenue NUMERIC(20,4) NOT NULL DEFAULT 0` 和 `net_amount NUMERIC(20,4)` —— 历史账单 `net_amount=NULL`，v1.2.0 后新生成的账单会回填。

## 「双向并网点」如何建模

物理一个电表（双向计量），软件层面拆成两条 `meters`：
- 一条 `role=GRID_TIE` `flow_direction=IMPORT`（电网买电）
- 一条 `role=GRID_TIE` `flow_direction=EXPORT`（上网卖电）

两条 meter 各自的 InfluxDB tag 不同，KPI 公式按 direction 分别取数。这样避免引入「正负值」语义，所有读数仍是 ≥ 0。

## KPI 公式（PV 启用时）

`总用电 = Σ(GRID_TIE.IMPORT) − Σ(GRID_TIE.EXPORT) + Σ(GENERATE.production)`

含义：电网净进口 + 自家光伏发电 = 厂区真实用电。`ems.feature.pv.enabled=false` 时短路到老路径 `Σ(CONSUME)`，与 v1.1.x 完全一致。

## 月账单计算

`feed_in_revenue` 计算方式：按小时遍历 `feed_in_tariff` 时段（SHARP/PEAK/FLAT/VALLEY），用每小时的 `EXPORT` 读数 × 当时段单价后求和。该值在月底批账时由 `BillingServiceImpl` 写入 `bill.feed_in_revenue` 和 `bill.net_amount`。

`SavingsPanel`（前端「账单」页面）直接展示三栏：月度电费 / 上网卖电收入 / 电费净额。

## 碳排报表

`reduction_kg = self_consumption_kwh × (carbon_factor[GRID] − carbon_factor[SOLAR])`

反映 PV 自发自用替代电网用电产生的减排当量。`CarbonPanel`（前端「报表」页面）展示自发自用 / 电网因子 / 减排当量三栏。

## 老客户行为承诺

未配置光伏的工厂在升级后行为完全不变：
- KPI 公式自动短路到老路径（`ems.feature.pv.enabled=false` 时）
- 月账单 `feed_in_revenue=0`、`net_amount=amount`
- 前端面板「能源来源构成」/「PV 发电 vs 负载」/「节费」/「碳排」均自动隐藏（依赖 `/api/v1/features` 返回的 `pv` 标志）

## 总闸控制

平台运维可通过修改 `application.yml` 中 `ems.feature.pv.enabled=false` 关闭整套 PV 路径，重启栈秒级生效（紧急回退用）。

`GET /api/v1/features` 返回当前总闸状态，前端据此条件渲染。

## 升级步骤（部署侧）

1. 部署 v1.2.0 镜像，`ems.feature.pv.enabled=false`（默认即关）
2. 跑 `V2.6.0` + `V2.6.1` migration（应用启动时自动）
3. 单租户灰度切 `enabled=true`，注入测试 PV 数据验证四面板
4. 全量放开
